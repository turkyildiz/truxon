Owner's "crazy idea" (2026-07-21): tablets should need ONE app. The side apps were **Tailscale** (VPN so the radio could reach the tailnet server) and **Mumla** (GPLv3 Mumble client → embedding it would GPL the whole app).

**Architecture shipped:** PTT voice over **Supabase Realtime broadcast** on private topic `radio:fleet` — no Mumble server, no VPN, works on any LTE (443/TLS), auth = Truxon logins. RLS policies on `realtime.messages` (migration 20260721040001) are the door; `my_role()` raise kills a deactivated login's radio. Wire format (`mobile/lib/services/radio_codec.dart`, mirrored in `frontend/src/pages/Radio.tsx`): event `ptt` `{u, q, c:[b64 opus 20ms frames]}`, 48kHz mono, 5 frames/message (~10 msg/s), event `state` `{u, on}`. Mobile: `record` (PCM stream) + `opus_dart`/`opus_flutter` (BSD) + `flutter_pcm_sound`; channel `self: true` (own-mic echo filtered by username). Web dispatch console at `/radio` uses WebCodecs (Chrome-only). Gradle needed `android.ndk.suppressMinSdkVersionError=21` (opus_flutter_android declares minSdk 19; app floor is 24).

**Forest on the net:** hold "🌲 Ask Forest" → on-device STT → `trux-agent` with `radio: true` (channelNote makes replies 1-3 spoken sentences) → reply TTS'd (British voice) → WAV parsed + resampled to 48k (pure-Dart in radio_codec) → broadcast as user `🌲 Forest`, paced real-time. The ASKING TABLET does all the work; zero server-side audio.

**Field-test findings (2026-07-21, owner in yard) + fixes shipped:**
- PTT cut off ~2s in: the hold button was a TAP GESTURE — Flutter cancels a tap when the finger drifts past ~4mm slop. Fixed with raw `Listener` onPointerDown/Up/Cancel (v+6). NEVER use GestureDetector tap events for hold-to-talk.
- Transient Supabase Realtime "missing partition" server error can wedge a channel as `off` — Retry/app-restart clears it; channel-error handler must null the channel so the next attempt rebuilds clean.
- **ALWAYS-ON RX (v+7/+8): `mobile/lib/services/radio_rx.dart`** runs in the GPS foreground-service isolate (flutter_foreground_task): standalone RealtimeClient (NO Supabase.instance there), opus decode + pcm playback, presence-tracks the driver. Radio plays with screen off/app swiped away. UI RadioService MUTES its own playback while prefs `radio_rx_alive` heartbeat is fresh (<150s) — no double audio; TX stays UI-side. Username handoff via prefs `radio_user` (home_shell writes at login). Manifest: foregroundServiceType `location|mediaPlayback` + FOREGROUND_SERVICE_MEDIA_PLAYBACK.
- **AUTH ARCHITECTURE CHANGE (v+8): `mobile/lib/services/auth_refresher.dart` is the ONLY refresh-token spender.** supabase_flutter autoRefreshToken is DISABLED in main.dart. Both isolates call `AuthRefresher.ensureFresh()` (claim-and-settle SharedPreferences lock, refresh via REST, writes back to supabase_flutter's own `sb-<ref>-auth-token` key; UI adopts via recoverSession only when the stored session is still live). This removed the ~1h app-closed ceiling — radio+GPS run indefinitely. NEVER call auth.refreshSession() directly anywhere in the app; that reintroduces the rotation-race logout bug.
- v+9: UI refresh — `mobile/lib/theme.dart` is the design system (tonal M3, auto dark mode, 48px targets, StatusPill); restyle by editing that one file.
- **v+11 — closed-app radio ACTUALLY works now; three emulator-verified bugs:** (1) **envelope shape**: a STANDALONE RealtimeClient's onBroadcast delivers the whole `{type,event,payload:{...}}` envelope while the supabase_flutter-configured socket delivers the inner payload — RadioRx discarded every voice packet silently (`payload['u']` null → unpackPtt null). Receiver accepts both shapes; remember this whenever using raw RealtimeClient in an isolate. (2) presence one-shot-track raced the radio_user pref after OTA service restarts → re-track every tick until named. (3) `Geolocator.getCurrentPosition` without `timeLimit` hangs FOREVER in dead-GPS spots and onStart awaits the first sample → repeat timer never started (froze GPS flush + radio ticks) — bounded 30s. Verified: external client sees service presence; ptt delivered+decoded with app killed.
